Predix UI

Predix UI provides a collection of reusable components to help you build your user friendly IoT web application quickly. With Predix’s plug-and-play components, you only have to add them to your page, like any HTML element and it is rendered without you worrying about what’s going on under the hood.

What are Predix Components?

Predix UI components, or px-components are Polymer web components that provide a nice user interface without the developer having to worry about the complexity. These components are reusable and can be used together to create powerful, user-friendly web applications. The Predix UI home has great documentation on how to use the components with examples.

Learn more about Predix UI Components here

Predix WebApp Starter

The Predix WebApp Starter has a backend implemented as NodeJS server, with session and proxy configurations, and examples. You can customize this or replace it with another server application, like Nginx. We'll not discuss about that here. Our main focus is on the front-end. We'll stick with the backend of the Webapp Starter.

The front-end of the Predix WebApp Starter is built using the Polymer library with Inuit CSS and encourages developers to extend it into a full-fledged web application. You can learn more about the Predix WebApp Starter here.

Using the Boiler Plate

This boiler plate application is aimed to give you a starting point to build your application using any front-end javascript framework/library, any CSS framework to design the look and feel you like. You can plug in any Predix UI component in this application because the px-components are framework agnostic. You don't have to build your application using Polymer because the components are built using Polymer.

For instance, if you want to include a px-card in your application, simply install it and add it your bower.json file by running 
bower install px-card --save
Import it in your application 
<link rel="import" href="bower_components/px-card/px-card.html>
And start using it 
<px-card header-text="Hello World!">...</px-card>

Running the application

Clone the boiler-plate branch from this repo

git clone -b boiler-plate https://github.com/abhishekdhaduvai/predix-seed.git

You will need node, bower, and gulp to be installed on your machine. If you don't already, you can download and install node from here. Run the following commands after you install node on your machine.

npm install bower -g
npm install gulp -g

Change directory to predix-seed and run the following commands.

npm install
bower install

Run the following command to start a local server. Your application will run on http://localhost:5000

gulp

Building an Angular App from the boiler plate

AngularJS is a popular front-end framework developed and maintained by Google. Its a great framework built for CRUD applications, which is what most IoT web-applications are. If you're looking to build a sleek, user-friendly Single-page applications, AngularJS is a good fit.

Running the application

Clone the angularjs branch from this repo.

git clone -b angularjs https://github.com/abhishekdhaduvai/predix-seed.git

The steps to launch the app are the same as above. Run the following commands.

npm install
bower install
gulp

Pushing the app to the cloud

When you're ready to push your application to CloudFoundry, update the manifest.yml file. Change the name of the application to yours and uncomment and add any services you want to bind to your application. Uncomment the clientId and base64ClientCredential and enter the values from your UAA.

---
applications:
  - name: predix-ui-seed
    memory: 64M
    buildpack: nodejs_buildpack
    command: node server/app.js
#services:
 # - -secure-uaa-instance
 # - -timeseries-instance
 # - -asset-instance
env:
    node_env: cloud
    uaa_service_label : predix-uaa
    # Add these values for authentication in the cloud
    #clientId: {Enter client ID, e.g. app-client-id, and place it here}
    #base64ClientCredential: dWFhLWNsaWVudC1pZDp1YWEtY2xpZW50LWlkLXNlY3JldA==
gulp dist
cf push
